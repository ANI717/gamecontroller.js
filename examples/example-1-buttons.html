Example: Buttons

Connect a gamepad to the computer, and click on the different buttons. They will highlight as they are pressed.

This is a 17-button standard gamepad layout as defined on the W3C Gamepad API definition. It may not match the gamepad that is connected, but it can be used to see how the buttons in that gamepad match the buttons in the standard gamepad.

Although in this library, the joystick/axes are handled as buttons, in this particular demo we are going to focus exclusively on buttons. Visit the second example to see how the joysticks/axes work.

How it works

Event handlers can be associated to the buttons using the .on() method:

gamepad.on(BUTTON_NAME, CALLBACK);

Only one action is allowed by button. If you use several .on() with the same button, the latest call to it will be the one that is applied when the button is pressed. 

gameControl.on('connect', function(gamepad) {
  gamepad.on('select', function() {
    // do something
  });
});

Some buttons have aliases, so it is easier to associate events to them. For example, we can add an event handler to the Select/Back by doing one of these two: 

gamepad.on('button8', function() {
  // do something
});

gamepad.on('select', function() {
  // do something
});

Using aliases doesn't mean that you will be able to use more than one action, as they are the same.

The available button aliases are:

  • select: equivalent to button8. (Works for Select/Back in some gamepads)
  • start: equivalent to button9. (Works for Start/Forward in some gamepads)
  • power: equivalent to button16. It is the power button available on some gamepads (e.g. the XBox logo button)
